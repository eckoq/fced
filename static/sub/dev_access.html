接入说明

本文档旨在帮助厂商在资源接入过程中，了解平台支持信息、支持的DOCKER镜像和接入流程，便于厂商们接入。

CPU架构和系统

可支持CPU架构: Arm32/Arm64/x86/x86_64
可支持操作系统：centos/ubuntu
备注： 其他CPU架构和操作系统暂不支持，有需要可以联系相关接入人员

网络环境

网络类型：矿机设备稳定全锥型或者静态IP；盒子设备要求对称型设备小于总数15%
域名解析：支持DNSServer 182.254.116.116

镜像支持

平台通过docker镜像的方式提供程序包，需要厂商设备可以支持docker
docker版本>=18.09
平台支持程序包镜像：
操作系统 Arm32 Arm64 x86 x86_64
centos7 N N Y Y
ubuntu19.10 Y Y N N

推荐配置

50Mbps以下主机配置

硬件 要求 是否必须 备注
CPU Arm 2核或者x86 1核2线程 是 最低要求
内存 128MB 是 最低要求
磁盘 75GB 是 最低要求，优先ssd
网卡 50Mbps 是 最低要求

50Mbps-200Mbps主机配置

硬件 要求 是否必须 备注
CPU Arm 4核或者x86 2核4线程 是 最低要求
内存 512MB 是 最低要求
磁盘 300GB 是
网卡 200Mbps 是

200Mbps-500Mbps主机配置

硬件 要求 是否必须 备注
CPU x86 4核8线程 是 主频>2GHZ，最低要求
内存 1GB 是 最低要求
磁盘 500GB 是
网卡 500Mbps 是

500Mbps-1Gbps主机配置

硬件 要求 是否必须 备注
CPU x86 8核16线程 是 主频>2GHZ，最低要求
内存 2GB 是 最低要求
磁盘 1TB 是
网卡 1Gbps 是

10Gbps主机配置

硬件 要求 是否必须 备注
CPU x86 16核32线程 是 主频>2GHZ，最低要求
内存 32GB 是
磁盘 15TB 是
网卡 10Gbps 是

磁盘配置

磁盘128K随机读能力： SSD读带宽1.5Gbps，HDD读带宽100Mbps (SSD和HDD的128KB随机读带宽为平台提供的建议值，厂商可用磁盘实测数据为准)
磁盘总存储容量：设备1Gbps上行带宽需要磁盘总存储1.5TB，部分设备可参考《推荐配置》
磁盘要求：

  • 磁盘128K随机读能力 >= 网卡上行能力。 如网卡为3Gbps设备，则需要2块SSD才可以保证1.5Gbps*2 >= 3Gbps
  • 磁盘总存储容量 >= 配置推荐值。 如网卡为3Gbps设备，需要4.5TB存储。2块SSD保证磁盘读性能，如果SSD为1TB，可外挂1块HDD。

厂商接入流程

step 1 厂商整理设备基本信息和和配置的情况说明，进入接入信息，下载厂商接入信息表，参考介绍填写表格

step 2 根据厂商提供的《接入信息》表，双方接口人沟通是否符合要求

step 3 设备测试验证，厂商提供测试设备，部署平台提供的程序镜像

step 4 设备验收，双方沟通带宽需求后厂商提供批量。进入设备验收，下载设备验收信息表，参考介绍填写表格

step 5 设备部署，平台安排设备部署上线和放量计划
